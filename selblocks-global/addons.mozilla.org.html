Summary

It's enhanced SelBlocksGlobal. It works in Selenium IDE.

Detailed description

It's an enhanced version of SelBlocks. It's just like SelBlocks, but it allows to call 'scripts' across the test cases (within the same test suite). For documentation of Selenese see the original summary of SelBlocks extension itself and its original documentation. Since you are here, you'd also benefit from other extensions of SeLite family, which allow DB-driven automated tests. The only backward incompatibility when migrating from SelBlocks: SelBlocksGlobal defines command 'breakLoop' instead of 'break', because 'break' is a standard Selenese command. You'll need Se TestcaseDebugContext and Selenium IDE (tested with version 1.10.0 and 2.0.0).
