Summary

It reloads your Selenium Core extension (JS) automatically, without a need to restart Selenium IDE. It loads your custom JS before executing the first test command/test case/test suite. If the JS gets modified, it is re-loaded automatically.

Detailed description

It reloads your Selenium Core extension javascript automatically, without a need to restart Selenium IDE. It loads your custom javascript file before executing the first test command/test case/test suite. If the javascript file is modified, it gets re-loaded automatically next time you
  • run a single Selenese command (before that command)
  • run a test case/test suite (before the case/suite)
  • pause and subsequently resume a test case/test suite (before it is resumed)
Requirements and limitations
  • Non-strict javascript: If you want to 'export' any new top-level variables/functions then do not have "use strict"; at the file level.
  • Functions to 'export'
    • Define functions using myFunctionName= function( params... ) { body... };
    • Do not use function myFunctionName( params... ) { body... } because that won't be visible.
    • If you use function myFunctionName( params... ) { body... } and then you change it to use myFunctionName= function( params... ) { body... }; that will have no effect - you need to reload whole Selenium IDE.
  • Variables to 'export'
    • use variableName= ...;
    • don't use var variableName= ...;
    • if you define a variable using var variableName= ...; and then you change it not to use 'var', that will have no effect - you need to reload whole Selenium IDE
  • Adding/modifying Selenese (commands/getters etc.)
    • If you introduce a new Selenese command/getter - Selenium.prototype.doXXX, Selenium.prototype.getXXX, then those will be available to your test case/test suite only *after* you run an (any) Selenese command first. If you modify any existing Selenese command/getter, that will have effect when your run any next Selenese.
    • If you tail-intercept existing an method, then save the original method on the first load only. Don't increase the intercept chain each time the script is run. Use for example: if( (typeof originalMethod)==="undefined" ) { var originalMethod= Selenium.prototype.methodName; } Selenium.prototype.methodName= function(pqr...) { originalMethod.call(this); // extra new tasks... }
  • SeBootstrap doesn't load your custom file at the start of Selenium IDE (if you've selected the file previously), but only after you run the first (any) Selenese action. Otherwise any errors from your custom script (and actions/getters definied in it) couldn't log to Selenium IDE log console. So don't have your first Selenese command depend on your bootstrapped script.
You'll need Selenium IDE (tested with version 1.10.0 and 2.0.0).
